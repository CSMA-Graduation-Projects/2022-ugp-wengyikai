# 故障排除指南

## 常见问题及解决方案

### 后端启动问题

#### 问题1：ModuleNotFoundError: No module named 'xxx'

**症状**: 
```
ModuleNotFoundError: No module named 'fastapi'
```

**原因**: 依赖未安装

**解决**:
```bash
cd backend
pip install -r requirements.txt

# 如果仍有问题，重新创建虚拟环境
python -m venv venv
# Windows
venv\Scripts\activate
# macOS/Linux
source venv/bin/activate

pip install -r requirements.txt
```

---

#### 问题2：端口被占用

**症状**:
```
ERROR: [Errno 48] Address already in use
```

**原因**: 8000 端口已被其他程序占用

**解决**:

方案 A - 查找并关闭占用程序：
```bash
# Windows
netstat -ano | findstr :8000
taskkill /PID <PID> /F

# macOS/Linux
lsof -i :8000
kill -9 <PID>
```

方案 B - 使用不同端口：
```bash
uvicorn main:app --port 8001
```

---

#### 问题3：数据库连接失败

**症状**:
```
sqlalchemy.exc.OperationalError: (sqlite3.DatabaseError) database disk image is malformed
```

**原因**: 数据库文件损坏或权限问题

**解决**:
```bash
# 备份原数据库
cp user_story_kb.db user_story_kb.db.bak

# 删除坏的数据库
rm user_story_kb.db

# 重新启动，会自动创建新数据库
python main.py
```

---

#### 问题4：API 密钥无效

**症状**:
```
DEEPSEEK_API_KEY not found in environment
```

**原因**: 未设置环境变量

**解决**:

1. 复制示例文件
```bash
cp .env.example .env
```

2. 编辑 `.env` 文件，填写实际的 API 密钥
```env
DEEPSEEK_API_KEY=sk_xxxxxxxxxxxx
```

3. 重启应用
```bash
python main.py
```

### 前端连接问题

#### 问题5：前端无法连接后端

**症状**: 
```
Failed to load resource: the server responded with a status of 404
GET http://localhost:8000/api/v1/generate_user_stories 404
```

**原因**: 后端未启动或地址配置错误

**解决**:

1. 检查后端是否运行
```bash
curl http://localhost:8000/health
```

2. 如果后端未运行，启动后端
```bash
cd backend
python main.py
```

3. 检查前端的 API 地址配置
```javascript
// src/config.js 中检查
const API_BASE_URL = process.env.VUE_APP_API_URL || 'http://localhost:8000/api/v1'
```

4. 如果使用 Docker，确保容器间网络连通
```bash
docker-compose logs backend
docker-compose logs frontend
```

#### 问题6：CORS 跨域错误

**症状**:
```
Access to XMLHttpRequest at 'http://localhost:8000/api/v1/...' from origin 'http://localhost:8080' 
has been blocked by CORS policy
```

**原因**: 后端 CORS 配置不允许前端域名

**解决**:

```python
# backend/main.py 中修改 CORS 配置
app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],  # 开发环境可以用 "*"
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)

# 或者指定具体域名
app.add_middleware(
    CORSMiddleware,
    allow_origins=[
        "http://localhost:8080",
        "http://127.0.0.1:8080"
    ],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

重启后端生效。

### 功能问题

#### 问题7：生成用户故事为空

**症状**: API 返回空列表或没有生成任何故事

**原因**: 
- LLM API 调用失败
- 输入需求格式不正确
- LLM 服务暂时不可用

**解决**:

1. 检查网络连接
```bash
ping api.deepseek.com
```

2. 验证 API 密钥
```bash
python -c "import os; print(os.getenv('DEEPSEEK_API_KEY'))"
```

3. 查看详细日志
```python
# 在 main.py 中启用日志
import logging
logging.basicConfig(level=logging.DEBUG)
```

4. 尝试更清晰的需求输入
```
不推荐: "我需要一个系统"
推荐: "我需要开发一个在线购物平台，用户可以浏览商品、添加购物车、下单支付"
```

#### 问题8：生成代码语法错误

**症状**: 生成的代码无法运行

**原因**: AI 生成的代码不完整或有错误

**解决**:

1. 生成的代码只是框架，需要完善
```python
# 检查生成的代码
code = api_response['data']['backend_code']

# 验证语法
import ast
try:
    ast.parse(code)
except SyntaxError as e:
    print(f"语法错误: {e}")
```

2. 提供更详细的需求
```
不好: "生成登录代码"
更好: "使用 FastAPI + SQLAlchemy 生成用户登录接口，包含密码加密和错误处理"
```

3. 手工调整和补完生成的代码

#### 问题9：知识库查询失败

**症状**:
```
No knowledge module found for: xxx
```

**原因**: 知识库中不存在该模块

**解决**:

1. 查看可用的知识模块
```bash
curl http://localhost:8000/api/v1/kb/statistics
```

2. 添加新的知识模块
```bash
curl -X POST http://localhost:8000/api/v1/kb/add_module \
  -H "Content-Type: application/json" \
  -d '{
    "module_name": "新模块",
    "category": "分类",
    "description": "描述"
  }'
```

3. 使用近似的模块名
```
# 知识库包含: "用户登录"
# 你可以这样问: "添加登录功能"
```

### Docker 部署问题

#### 问题10：Docker 构建失败

**症状**:
```
failed to solve with frontend dockerfile.v0
```

**原因**: Docker 镜像构建错误

**解决**:

```bash
# 清理缓存
docker system prune -a

# 查看详细构建日志
docker compose build --no-cache --verbose

# 逐个构建检查
docker build -t user-story-backend ./backend --progress=plain
docker build -t user-story-frontend ./frontend --progress=plain
```

---

#### 问题11：Docker 容器无法访问

**症状**: 访问 `http://localhost:8080` 超时

**原因**: 容器未正确启动或端口映射有误

**解决**:

```bash
# 查看容器状态
docker compose ps

# 查看容器日志
docker compose logs backend
docker compose logs frontend

# 检查网络连接
docker compose exec frontend ping backend

# 重启容器
docker compose restart

# 完全重建
docker compose down -v
docker compose up -d --build
```

---

#### 问题12：数据库卷持久化失败

**症状**: 重建容器后数据消失

**原因**: 卷配置错误

**解决**:

```yaml
# docker-compose.yml
services:
  backend:
    volumes:
      - backend-storage:/app/storage  # 确保卷路径正确

volumes:
  backend-storage:
    driver: local
```

```bash
# 检查卷信息
docker volume ls
docker volume inspect userstoryllm_backend-storage

# 备份数据
docker run --rm -v userstoryllm_backend-storage:/storage \
  -v $(pwd):/backup busybox \
  cp -r /storage /backup/storage_backup

# 恢复数据
docker run --rm -v userstoryllm_backend-storage:/storage \
  -v $(pwd):/backup busybox \
  cp -r /backup/storage_backup /storage
```

### 性能问题

#### 问题13：生成速度很慢

**症状**: 请求需要很长时间才能返回

**原因**: 
- LLM API 响应慢
- 网络连接不好
- 知识库查询慢

**解决**:

```bash
# 检查网络延迟
ping api.deepseek.com

# 启用性能追踪
export ENABLE_PERFORMANCE_TRACE=true

# 查看各步骤耗时
curl http://localhost:8000/api/v1/generate_user_stories \
  -H "Content-Type: application/json" \
  -d '{"requirement": "test"}' | jq '.data'
```

**优化建议**:

```env
# 增加超时时间
LLM_TIMEOUT=60

# 启用缓存
LLM_ENABLE_CACHE=true

# 减少 MAX_TOKENS（如果允许）
LLM_MAX_TOKENS=2000
```

#### 问题14：内存占用过高

**症状**: 应用运行一段时间后内存持续增长

**原因**: 
- 缓存未清理
- 数据库连接未关闭
- 内存泄漏

**解决**:

```python
# 检查数据库连接
from sqlalchemy import event
from sqlalchemy.pool import Pool

@event.listens_for(Pool, "connect")
def receive_connect(dbapi_conn, connection_record):
    print(f"DB Pool size: {connection_record.info}")
```

```bash
# 监控内存使用
docker stats backend

# 限制容器内存
docker compose.yml:
  services:
    backend:
      mem_limit: 1g
      memswap_limit: 2g
```

## 快速诊断工具

### 健康检查脚本

```bash
#!/bin/bash

echo "=== 系统检查 ==="
echo "1. 检查 Python 版本"
python --version

echo "2. 检查后端服务"
curl -s http://localhost:8000/health || echo "后端未运行"

echo "3. 检查前端服务"
curl -s http://localhost:8080 || echo "前端未运行"

echo "4. 检查数据库"
sqlite3 backend/user_story_kb.db ".tables" 2>/dev/null || echo "数据库损坏"

echo "5. 检查环境变量"
env | grep -E "DEEPSEEK|DATABASE" || echo "环境变量未设置"

echo "=== 检查完成 ==="
```

### 日志分析

```bash
# 查看最近的错误
tail -f logs/app.log | grep ERROR

# 搜索特定错误
grep "ModuleNotFoundError" logs/app.log

# 性能分析
grep "generation_time" logs/app.log | awk '{sum+=$NF; count++} END {print "平均耗时: " sum/count " ms"}'
```
