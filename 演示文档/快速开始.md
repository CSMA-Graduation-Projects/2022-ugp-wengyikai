# UserStoryLLM 快速开始指南

## 什么是 UserStoryLLM？

UserStoryLLM 是一个基于大语言模型的智能用户故事生成系统。它可以帮助您：
- 从自然语言需求自动生成结构化的用户故事
- 将用户故事自动拆解为具体的任务
- 基于用户故事生成相关代码框架
- 通过知识库增强需求理解

## 核心功能

### 1. 需求转用户故事
将自然语言需求转换为标准的用户故事格式：
```
输入: "用户需要一个登录功能"
输出: 
- 用户故事1: 作为新用户，我想快速创建账号，以便访问系统
- 用户故事2: 作为已注册用户，我想输入用户名和密码登录，以便使用系统功能
- 用户故事3: 作为系统管理员，我想管理用户权限，以便控制访问权限
```

### 2. 故事任务拆解
将用户故事自动拆解为具体的任务：
```
用户故事：作为在线购物用户，我想快速完成支付，以便尽快获得商品

拆解后的任务：
- 后端任务：设计订单表和支付表
- 后端任务：实现支付接口和状态查询
- 前端任务：设计支付结果页面
- 前端任务：实现订单确认和支付流程
```

### 3. 知识库增强
系统内置知识库包含常见的系统模块，如：
- 登录/注册
- 用户管理
- 权限管理
- 商品管理
- 购物车
- 订单支付
- 评论评分
- 等等...

## 使用步骤

### 方式一：本地开发环境

1. **安装依赖**
```bash
cd backend
pip install -r requirements.txt
cd ../frontend
npm install
```

2. **启动后端**
```bash
cd backend
python main.py
```

3. **启动前端**（新终端）
```bash
cd frontend
npm run serve
```

4. **访问应用**
- 前端地址：http://localhost:8080
- 后端 API：http://localhost:8000

### 方式二：Docker 一键部署

```bash
# 复制环境配置文件
cp .env.example .env

# 构建并启动服务
docker compose up -d --build

# 访问应用
# 前端：http://127.0.0.1:8080
# 后端健康检查：http://127.0.0.1:8080/api/health
```

## 常见任务

### 任务1：生成用户故事
1. 在输入框中输入需求，例如："我需要一个用户评论功能"
2. 点击"分析需求"按钮
3. 系统会显示生成的用户故事列表
4. 可以查看每个故事的详细信息

### 任务2：拆解用户故事为任务
1. 选择一个用户故事
2. 点击"拆解任务"按钮
3. 系统会将故事拆解为前端/后端具体任务
4. 显示任务优先级和依赖关系

### 任务3：生成相关代码
1. 选择用户故事和对应任务
2. 点击"生成代码"
3. 系统生成代码框架和实现建议
4. 支持 Python、TypeScript 等多种语言

### 任务4：从代码提取需求
1. 输入现有代码
2. 点击"从代码提取需求"
3. 系统分析代码并生成对应的需求描述
4. 可用于反向工程和文档生成

## API 端点示例

### 1. 生成用户故事
```bash
curl -X POST "http://localhost:8000/api/v1/generate_user_stories" \
  -H "Content-Type: application/json" \
  -d '{"requirement": "我需要一个支付功能"}'
```

### 2. 拆解任务
```bash
curl -X POST "http://localhost:8000/api/v1/generate_tasks" \
  -H "Content-Type: application/json" \
  -d '{"story": "作为购物用户，我想快速完成支付，以便及时获得商品"}'
```

### 3. 生成代码
```bash
curl -X POST "http://localhost:8000/api/v1/generate_code" \
  -H "Content-Type: application/json" \
  -d '{
    "requirement": "支付功能",
    "story": "作为购物用户，我想快速完成支付",
    "tasks": [{"task": "实现支付接口"}]
  }'
```

## 环境配置

### 必需的环境变量

`.env` 文件中需要配置：

```env
# API 密钥
DEEPSEEK_API_KEY=your_api_key_here

# 数据库连接
DATABASE_URL=sqlite:///user_story_kb.db

# 应用端口
APP_PORT=8080

# 大模型配置
LLM_MODEL=deepseek-chat
LLM_TEMPERATURE=0.7
```

## 故障排除

### 问题1：后端无法启动
- 检查 Python 版本是否 >= 3.10
- 确认依赖已安装：`pip install -r requirements.txt`
- 检查端口 8000 是否被占用

### 问题2：前端无法连接后端
- 确保后端已启动
- 检查 CORS 跨域配置
- 在浏览器控制台查看网络错误

### 问题3：Docker 构建失败
- 清理缓存：`docker compose down -v`
- 重新构建：`docker compose build --no-cache`
